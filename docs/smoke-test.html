Phase 1 smoke test

First end-to-end live run of audio-learn podcast against real Claude + Gemini.
Checkbox state is saved to localStorage — close the tab and come back.

1Install system prerequisites

  • python3 --version   # expect 3.11+

    If your default python3 is older (macOS often ships 3.9), use a newer one explicitly: python3.13 -m venv .venv.

  • brew install ffmpeg          # macOS
sudo apt install ffmpeg      # Debian/Ubuntu
which ffmpeg                 # confirm on PATH

    pydub uses ffmpeg to encode PCM to MP3. Without it the TTS stage will fail at the encode step.

  • which claude
claude --version

    The research stage shells out to claude --print --allowedTools WebFetch,WebSearch,Read. Install instructions: docs.anthropic.com/claude/docs/claude-code.

  • Get one at aistudio.google.com/apikey (free tier is fine for one smoke run).

2Set up the repo

  • cd ~/Documents/projects/hubed-audio-learn
git status                   # should be clean, on main
source .venv/bin/activate    # venv from Task 1
pip install -e ".[dev]"      # idempotent — no-op if already installed
  • cp .env.example .env
# edit .env — at minimum set:
#   GEMINI_API_KEY=<your key>
# optionally point OUTPUT_DIR at your Syncthing/Obsidian folder:
#   OUTPUT_DIR=~/Podcasts/
    Never commit .env. It's gitignored, but if you use direnv, do not create .envrc with secrets either — .envrc is currently not in .gitignore (see review item #2).

3Verify configuration with doctor

  • audio-learn doctor

    Expected output:

    audio-learn version: 0.1.0
LLM provider:        claude
TTS provider:        gemini
OUTPUT_DIR:          /abs/path/output
WORK_DIR:            /abs/path/.work
GEMINI_API_KEY: set
claude binary: /opt/homebrew/bin/claude   (or similar)
ffmpeg: /opt/homebrew/bin/ffmpeg          (or similar)

    Exit code should be 0. If any line shows MISSING or NOT FOUND, fix that before continuing — those checks have to pass for the live run to succeed.

4Run the smoke episode

  • audio-learn podcast "Kalman filters" --length short

    Why short: ~1200 words, ~5–8 minutes of audio. Cheaper to fail fast on the first run; switch to medium later once it works end-to-end.

    You should see JSONL lines on stdout from each stage, e.g.:

    {"ts":..., "stage":"research", "event":"start", ...}
{"ts":..., "stage":"research", "event":"done", "data":{"elapsed_seconds":42.1}}
{"ts":..., "stage":"outline", "event":"start", ...}
...
Done. Manifest: /abs/path/output/2026-05-24-kalman-filters.manifest.json

    Total wall time on a first run: ~2–5 minutes (research is the slowest, ~30–90s; TTS is the next slowest).

    If research stage failed after 2 attempts: this is review finding #3 (Claude wraps JSON in markdown fences). Workaround: edit src/hubed_audio_learn/providers/llm/claude.py:43 to strip leading/trailing ```json fences before returning, or just open docs/review-2026-05-24.md §3 for the fix.
    If a stage hangs longer than 10 minutes: Ctrl-C, check stdout/stderr, and look in ./.work/<episode_id>/ for the last successfully-written JSON. The pipeline isn't resumable in Phase 1, so you'll restart from scratch.

5Inspect the artifacts

  • ls -la output/   # or your $OUTPUT_DIR

    Expect:

    2026-05-24-kalman-filters.mp3
2026-05-24-kalman-filters.html
2026-05-24-kalman-filters.transcript.md
2026-05-24-kalman-filters.outline.json
2026-05-24-kalman-filters.manifest.json
  • open output/2026-05-24-kalman-filters.mp3   # macOS
xdg-open output/...mp3                       # Linux

    Listen to 30s. You should hear Alex (curious) and Sam (explainer) alternating, no chopping, no garbled audio.

  • open output/2026-05-24-kalman-filters.html

    Play the audio. As each slide marker triggers, the corresponding box should get a blue border and scroll into view. Math figures should render via KaTeX (e.g., x̂_{k|k-1} = F_k x̂_{k-1|k-1} + B_k u_k).

    Click any slide — the audio should seek to that slide's first appearance.

    If slide-cue timing drifts: this is review finding #6 (silence-detection fallback). Open ./.work/<episode_id>/timing.json and eyeball whether the start_seconds values look reasonable vs. the actual audio.
  • less output/2026-05-24-kalman-filters.transcript.md

    Each block: [mm:ss] Host: line followed by the turn text. Timestamps should be monotonically increasing.

  • cat output/2026-05-24-kalman-filters.manifest.json | python -m json.tool

    Sanity-check: topic, providers: {llm: "claude", tts: "gemini"}, turn_count > 0, stage_durations_seconds for each of the 6 stages, at least one entry in sources (Claude found web sources during research).

7What to do with what you learn

  • Things that are most likely to bite first (from the code review at docs/review-2026-05-24.md):

    1. Claude code-fence wrapping — if you hit it, file the patch from review §3.
    2. Slide-sync drift — note where in the audio it drifts and by how many seconds. Useful for review §6.
    3. Gemini cryptic errors — if TTS crashed with an IndexError, that's review §4. Capture the response shape.
    4. Re-run overwrite — if you re-ran the same topic the same day, the first episode is gone (review §5).

    If the smoke run worked cleanly, Phase 2 planning can prioritize features (PDF mode, OSS providers, Telegram bot) over the hardening items.
